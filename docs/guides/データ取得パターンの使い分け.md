# データ取得パターンの使い分け

受け取り側のコンポーネントの性質で判断する。

| 受け取り側 | パターン | 理由 |
|---|---|---|
| Server Component（表示のみ） | awaitして渡す | Server Componentのまま完結できる |
| Client Component（インタラクティブ） | promiseのまま渡す（`use()` API） | `<Suspense>`でfallbackを先に表示でき、ブロックしない |
| リアルタイム更新・ポーリングが必要 | SWR / React Query | サーバーで完結しないケース |
